# AGENTS.md

## 目标

将通过浏览器剪藏进入知识库的 markdown 原文，转换为适合长期复用的中文知识笔记。输出不追求逐句翻译，而追求：

- 去噪后的有效信息保留
- 结构化重写
- 可迁移的方法、概念与结论沉淀

## 适用范围

仅处理以下对象：

- 目录：`sources/`
- 文件类型：`.md`
- frontmatter 条件：`status: "new"`

输出位置：

- 目录：`codex/`
- 文件名：与原文件完全一致

## 工作原则

- 默认直接执行，不停留在泛泛分析
- 优先利用仓库内已有 `skills/` 中的技能说明，而不是每次从零设计流程
- 若技能与本文件冲突：具体技能优先，通用约束仍然生效
- 不做逐句翻译；以“知识重写”和“长期复用”为第一目标
- 不补写无法从原文、元数据或可访问来源中确认的事实

## 技能使用协议

当用户给出任务时，模型应主动检查是否存在可复用技能，并自行决定是否调用。

### 1. 技能发现

- 先查看 `skills/` 目录下的技能文件
- 技能文件默认为 `*.md`
- 至少读取与当前任务最相关的技能文件正文，而不是只看文件名

### 2. 技能匹配

按以下顺序判断是否使用技能：

1. 来源匹配：如站点、域名、平台、出版源明显匹配某个技能
2. 任务匹配：如“论文整理”“网页知识化”“摘要提炼”“方法归纳”等目标与技能说明一致
3. 内容特征匹配：如正文包含特定链接、术语、版式或元数据特征

### 3. 调用规则

- 一旦判断匹配，应先遵循对应技能中的流程、输出要求与质量标准
- 不必等待用户再次确认“是否使用该技能”
- 若技能内容不足以覆盖当前任务，再回退到本文件的通用流程补齐

### 4. 当前已知技能

- `skills/arXiv_skill.md`
  适用于 arXiv 摘要页、论文页，以及附带 arXiv `HTML` / `PDF` 链接的剪藏

### 5. 当前明确触发条件

满足以下任一条件时，优先调用 `skills/arXiv_skill.md`：

- 原文链接、`source`、正文链接中出现 `arxiv.org`
- 剪藏内容明确是 arXiv 论文摘要页或论文落地页
- 文中存在 arXiv 论文编号、`abs` 页、`pdf` 页、`HTML` 页等明显信号

## 标准执行流程

除非具体技能另有要求，否则按以下顺序执行：

1. 确认任务范围，仅处理 `sources/` 下 frontmatter `status: "new"` 的 `.md`
2. 如用户指定了文件、主题或批次，仅在指定范围内筛选目标
3. 检查 `skills/` 中是否有更具体的技能可用，并优先读取该技能
4. 读取原始 markdown，识别 frontmatter、来源链接、正文结构与噪声区块
5. 如原文中提供 `source`、`url`、`HTML`、`PDF` 等可访问来源，优先补读必要上下文
6. 去除导航、广告、推荐阅读、页脚、版权提示、重复段落等网页噪声
7. 提取可复用信息：问题、背景、概念、方法、步骤、证据、结论、边界、适用条件
8. 重写为中文知识笔记并写入 `codex/` 下同名文件
9. 如果任务要求闭环处理，则将原文件状态从 `new` 改为适当的已处理状态；若未要求改动原文件状态，至少在结果中说明是否已处理

## 输出写作要求

- title 不需要翻译保持原文，其他内容使用中文表达，保留必要英文术语、论文名、产品名、专有名词
- 以“重写”代替“翻译”，优先输出结构化结论
- 删除原网页中的口语化、宣传性、导航性内容
- 对信息来源层级保持敏感：
  - 摘要页只支持摘要级结论，不应伪装成全文级结论
  - 二手转述只写成“原文提到”或“可确认的信息”
- 信息不足时，明确写出“原文未提供”或“当前材料无法确认”
- 不新增未被原文支持的实验细节、数字、因果结论或作者意图

## 推荐笔记结构

```markdown
title: "MobileUse: A GUI Agent with Hierarchical Reflection for Autonomous Mobile Operation"
summary: 通过 action、trajectory、global 三层反思和主动探索机制，提升移动长程任务中的鲁棒执行能力
date: 2025-07-21
category: publication
badge: NeurIPS'26
authors:
  - Ning Li
  - Xiangmou Qu
  - Jiamu Zhou
  - et al.
tags:
  - Mobile Agent
  - GUI Agent
  - Reflection
  - Exploration
links:
  论文: https://arxiv.org/abs/2507.16853
  代码: https://github.com/MadeAgents/mobile-use

## ...
```


可根据原文类型裁剪结构，不要求机械保留所有小节，最好包含背景 / 技术问题 / 技术方案 / 实验 / 结论等内容

## 文件与状态约束

- 输入目录固定为 `sources/`
- 输出目录固定为 `codex/`
- 输出文件名必须与原文件完全一致
- 不处理 `status` 不是 `new` 的文件
- 不覆盖与当前任务无关的已有文件

## 失败与回退规则

- 若没有匹配技能：按本文件通用流程执行
- 若技能存在但说明不足：采用技能的核心约束，再用本文件补足
- 若原文损坏、frontmatter 缺失、来源不可访问或信息明显不足：输出保守版本，不自行脑补
- 若目标文件不在允许范围内：明确拒绝处理，并说明允许范围

## 完成标准

处理结果应同时满足：

- 相比原始剪藏，噪声显著减少
- 读者能快速把握主题、方法和结论
- 内容结构明显优于原文堆叠
- 结论与证据边界清楚，没有过度推断
- 结果可作为后续检索、引用与复用的中文知识笔记
- 内容不要口语化，不要用比喻等举例的方式
